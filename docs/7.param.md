# 7.参数

在机器人开发中，经常需要调整节点的运行`参数` —— 比如设置控制算法的PID比例系数是`10`还是`100`，图像处理算法需要检测`红色`还是`蓝色`，避障算法的安全距离参数是`0.3米`还是`0.5米`。我们希望这些`参数`在程序运行时`可查询`、`可修改`，甚至能`动态调整`，以适应不同场景。

ROS中的**参数（Parameter）机制**正是为此设计，它允许开发者以标准化的方式定义、访问和修改节点的参数信息，同时支持参数的动态更新、保存、加载等。所以，ROS中的**参数**就像节点的"旋钮"，不需要修改代码，就能改变节点的行为。

本章将介绍参数的使用方法，包括参数的基本概念、常用命令和基础代码实现。

## 7.1 参数的基本概念

### 7.1.1 什么是参数？

**参数**是节点运行时的配置变量，用于存储节点的静态或动态属性。它可以理解为节点的 "全局变量"，但具有更规范的访问方式和生命周期管理。例如：

+ 摄像头节点的image_width（图像宽度）、frame_rate（帧率）；
+ PID 控制器节点的kp（比例系数）、ki（积分系数）、kd（微分系数）；
+ 导航节点的max_speed（最大速度）、obstacle_distance（障碍物安全距离）。

参数与话题、服务的核心区别在于：**参数是节点自身的状态配置**，而非节点间的通信数据。话题和服务用于节点间的数据传递，而参数用于描述节点自身的运行属性。

### 7.1.2 参数的特点

+ **节点私有性**
    每个参数隶属于特定节点，默认情况下只能通过该节点的名称空间访问（如/turtlesim/background_r表示turtlesim节点的背景红色分量参数）。这种设计避免了参数命名冲突，确保配置的独立性。
+ **动态可配置性**
    参数的值可以在节点运行时被查询或修改，无需重启节点。例如，可通过命令行或代码实时调整 PID 控制器的参数，观察系统响应变化。
+ **支持多种数据类型**
    ROS2 参数支持常见的数据类型，包括：
    + 基本类型：bool（布尔值）、int64（整数）、float64（浮点数）、string（字符串）；
    + 复合类型：byte_array（字节数组）、bool_array（布尔数组）、int64_array（整数数组）、float64_array（浮点数数组）、string_array（字符串数组）。

+ **默认值与强制声明**
    节点可以为参数设置默认值，若未设置默认值则需在启动时指定（否则节点启动失败）。这种机制确保节点运行时拥有必要的配置信息。

+ **参数服务器与分布式访问**
    ROS2 通过参数服务器（Parameter Server）提供分布式参数访问能力，允许任意节点查询或修改其他节点的参数（需权限允许），简化了多节点系统的配置管理。

## 7.2 参数的命令行操作

ROS2 提供了简单的命令行工具，不用写代码就能查看和修改参数。我们用turtlesim节点来演示，先启动节点：

```
ros2 run turtlesim turtlesim_node
```

### 7.2.1 查看参数列表

想知道一个节点有哪些可调整的参数，用命令：

```
ros2 param list
```

会看到turtlesim节点的参数：

```
/turtlesim:
  background_b  # 背景蓝色分量（0-255）
  background_g  # 背景绿色分量（0-255）
  background_r  # 背景红色分量（0-255）
  ...
```

### 7.2.2 查看参数的值

想知道某个参数当前是多少，用`ros2 param get <节点名> <参数名>`：

```
# 查看背景红色分量的值
ros2 param get /turtlesim background_r
```

输出：

```
Integer value is: 69  # 当前红色分量是69
```

### 7.2.3 修改参数的值

想调整参数，用`ros2 param set <节点名> <参数名> <新值>`：

```
# 把背景红色分量改成255（最红）
ros2 param set /turtlesim background_r 255
```

此时海龟窗口的背景会立刻变红，不用重启节点 —— 这就是参数的 "动态调整" 能力。

### 7.2.4 保存和加载参数

如果调好的参数想下次用，可以保存到文件：

```
# 把turtlesim的参数保存到turtle_params.yaml
ros2 param dump /turtlesim > turtle_params.yaml
```

下次启动节点时，直接加载这个文件：

```
# 用保存的参数启动turtlesim
ros2 run turtlesim turtlesim_node --ros-args --params-file ./turtle_params.yaml
```

## 7.3 用代码操作参数

实际开发中，我们需要在节点里通过代码控制参数。下面用Python写一个简单的示例节点，演示如何声明、获取和响应参数。

### 7.3.1 创建功能包

先创建一个功能包（也可以用以前创建过的功能包）：

```
cd ~/ros2_ws/src
ros2 pkg create --build-type ament_python my_params --dependencies rclpy
```

### 7.3.2 编写参数节点代码

进入功能包的代码目录，创建`param_node.py`：

```
cd my_params/my_params
gedit param_node.py
```

写入以下代码:

```python
# 导入ROS2的核心库
import rclpy
from rclpy.node import Node

# 定义一个节点类，继承自Node
class ParamNode(Node):
    def __init__(self):        
        super().__init__('param_node')        
        # 1. 声明参数（带默认值）
        # 基本类型参数：整数、浮点数、布尔值、字符串
        self.declare_parameter('fps', 10) # 整数参数，默认值10        
        self.declare_parameter('pi', 3.14) # 浮点数参数：默认值3.14        
        self.declare_parameter('is_running', True) # 布尔参数：默认值True       
        self.declare_parameter('msg', "hello")  # 字符串参数：默认值"hello"
        # 复合类型参数：整数数组、浮点数数组
        self.declare_parameter('numbers', [1, 2])  # 整数数组
        self.declare_parameter('pid', [1.1, 2.2, 3.3])  # 浮点数数组        
        # 2. 先打印一次初始参数值
        self.print_params()        
        # 3. 设置定时器，每秒打印一次参数（方便观察修改后的变化）
        self.timer = self.create_timer(1.0, self.timer_callback)

    # 打印当前参数值的函数
    def print_params(self):
        # 获取参数值（指定返回类型，确保类型正确）
        fps = self.get_parameter('fps').value
        pi = self.get_parameter('pi').value
        is_running = self.get_parameter('is_running').value
        msg = self.get_parameter('msg').value
        numbers = self.get_parameter('numbers').value
        pid = self.get_parameter('pid').value
        # 打印到终端
        print(f"fps={fps}, pi={pi}, is_running={is_running}, msg={msg}, numbers={numbers}, pid={pid}")

    # 定时器回调：每秒执行一次
    def timer_callback(self):
        self.print_params()

def main(args=None):
    rclpy.init(args=args)  # 初始化ROS2
    node = ParamNode()     # 创建节点
    rclpy.spin(node)       # 让节点持续运行
    node.destroy_node()    # 关闭节点
    rclpy.shutdown()       # 退出ROS2

if __name__ == '__main__':
    main()
```

### 7.3.3 配置节点入口

打开功能包根目录的`setup.py`，找到`entry_points`，添加一行让 ROS2 能找到我们的节点：

```python
entry_points={
    'console_scripts': [
        # 格式："命令名 = 包名.文件名:主函数"
        "param_demo = my_params.param_node:main",
    ],
},
```

### 7.3.4 构建和运行

```
# 回到工作空间，构建功能包
cd ~/ros2_ws
colcon build --packages-select my_params # 只构建这一个功能包

# 运行节点
ros2 run my_params param_demo
```

节点启动后，会每秒打印一次参数。此时打开新终端，用命令修改参数：

```
# 修改count为20
ros2 param set /param_node count 20

# 修改msg为"hi"
ros2 param set /param_node msg "hi"

# 修改numbers为[3,2,1]
ros2 param set /param_node numbers [3,2,1]
```

节点终端会立刻显示参数变化。

## 7.4 常见问题

+ **参数名写错了怎么办？**

    会提示"参数不存在"，检查节点名和参数名是否正确（区分大小写）。

+ **修改参数后没反应？**

    可能是节点没注册回调函数，或者参数类型不对（比如给整数参数传了字符串）。

## 总结

参数是 ROS2 中管理节点配置的简单工具，核心用法包括：

+ 用`ros2 param list/get/set/dump`命令行操作；
+ 用函数`declare_parameter(<参数名>,<默认值>)`声明参数；
+ 用函数`get_parameter(<参数名>).value`获取参数；

参数适合存放需要调整的配置，和话题（传数据）、服务（做交互）配合，能让机器人系统开发与调试更加灵活。
