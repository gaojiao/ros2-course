# 12.机器人自动跟随

机器人感知让机器"看见"，机器人控制让机器"行动"，而机器人自动跟随（Robot Following）则是让机器人"会思考"——它能根据感知结果自动决策速度，实现对目标的自主跟踪。

本章将基于前面两章的内容，综合实现一个绿色目标自动跟随系统。机器人通过摄像头检测画面中的绿色色块，计算其中心位置与面积（即目标距离与偏移量的近似指标），并据此自动调整线速度与角速度，发布到`cmd_vel`话题，从而实现自动跟随。

通过本章学习，将掌握：

+ 感知与控制之间的数据闭环；
+ 从感知结果计算速度控制量；
+ 在ROS 2中订阅话题并实现自动控制逻辑；
+ 在虚拟环境中验证自动跟随功能。

## 12.1 自动跟随原理

自动跟随的核心是**闭环控制（Closed-loop Control）**。

机器人不断地：

1) 感知目标位置；
1) 根据目标偏移与大小计算控制量；
1) 调整速度；
1) 再次感知并修正动作。

形成如下闭环：

`视觉感知 → 目标位置与面积 → 控制算法 → 速度输出(cmd_vel) → 机器人运动 → 新的视觉输入`

我们可以将绿色色块的中心坐标(cx, cy)和面积(area)视为视觉传感器输出的状态量：

+ `cx`: 代表目标的水平位置；
+ `area`: 代表目标距离的近似值（面积越大，距离越近）。

<img src="./imgs/xq4-pro.jpg" style="zoom:40%">

控制目标是：

+ 让目标位于画面中央（cx ≈ 图像宽度/2）；
+ 保持目标大小恒定（即距离合适，area ≈ 目标期望值）。

因此，控制器可根据误差计算线速度和角速度：

+ 角速度 ∝ (图像中心 - cx)
+ 线速度 ∝ (目标面积目标值 - 当前面积)

## 12.2 系统总体结构

机器人自动跟随系统可划分为以下节点模块：

| 节点名称            | 功能               | 话题             |
| --------------- | ---------------- | -------------- |
| `color_detector`  | 视觉感知，检测绿色色块中心与面积 | 发布 `/color_info` |
| `auto_follower`      | 订阅感知结果，计算跟随速度    | 订阅`/color_info`<br>发布 `/cmd_vel`  |
| `robot_control` | 订阅速度命令，转换为底盘串口指令 | 订阅 `/cmd_vel`  |

系统结构图如下：

`摄像头 → color_detector → color_info → follower → cmd_vel → robot_control → 底盘执行`

## 12.3 话题消息格式

前面机器人感知部分，绿色检测节点发布消息代码：

```python
msg = String()
msg.data = f"{cx}, {cy}, {area}"
self.publisher_.publish(msg)
```

其发布的消息内容格式为`cx, cy, area`，每个数字用逗号分隔，即：

`data: '324, 240, 1520'`

为了让跟随节点能解析这些信息，我们需要提取出中心点 `(cx, cy)` 和面积 `area`，并将它们转换为控制量。

对于`'324, 240, 1520'`格式的字符串，回想一下Python中如何提取其中的内容？

分为以下步骤：

```python
s = '324, 240, 1520'
ls = s.split(',')    #以逗号为分隔符，转为字符串列表['324', ' 240', ' 1520']
cx = float(ls[0])    #将字符串'324'转为浮点数
cy = float(ls[1])    #将字符串' 240'转为浮点数
area = float(ls[2])  #将字符串' 1520'转为浮点数
```


进入`follow_demo/follow_demo`目录，并新建节点文件`auto_follower.py`：

```

```

### (1) 自动跟随节点代码

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import String
from geometry_msgs.msg import Twist

class AutoFollower(Node):
    def __init__(self):
        super().__init__('auto_follower')
        self.subscription = self.create_subscription(
            String, 'color_info', self.listener_callback, 10)
        self.publisher_ = self.create_publisher(Twist, 'cmd_vel', 10)

        # 参数配置
        self.image_width = 640      # 图像宽度
        self.target_area = 20000     # 期望面积（距离控制目标）
        self.k_linear = 0.0002      # 面积误差比例系数
        self.k_angular = 0.002      # 水平误差比例系数

        self.get_logger().info("Auto Follower Node Started")

    def listener_callback(self, msg):
        ls = msg.data.split(',')    #以逗号为分隔符，转为字符串列表
        cx = float(ls[0])    #将字符串转为浮点数
        cy = float(ls[1])    #将字符串转为浮点数
        area = float(ls[2])  #将字符串转为浮点数
        # 绿色面积太小
        if area < 100:
            self.stop_robot()
            return

        # 计算误差
        err_x = (self.image_width / 2) - cx
        err_area = self.target_area - area
        # 计算速度（比例控制）
        angular_z = self.k_angular * err_x
        linear_x = self.k_linear * err_area
        # 构建并发布Twist消息
        twist = Twist()
        twist.linear.x = linear_x
        twist.angular.z = angular_z
        self.publisher_.publish(twist)

        self.get_logger().info(f"跟随控制: cx={cx}, area={area}, linear={linear_x:.2f}, angular={angular_z:.2f}")

    def stop_robot(self):
        twist = Twist()
        self.publisher_.publish(twist)
        self.get_logger().info("未检测到目标，停止运动")

def main(args=None):
    rclpy.init(args=args)
    node = AutoFollower()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### (2) 修改`setup.py`

在`setup.py`中添加：

```python
    'console_scripts': [
        'auto_follower = follow_demo.auto_follower:main',

```

+ 从`视觉检测 → 自动跟随 → 底盘控制`的完整流程：

    `$ socat -d -d pty,raw,echo=0 pty,raw,echo=0`
  + 然后启动感知、自动跟随、控制节点
    ```
    $ ros2 run perception_demo color_detector
    $ ros2 run follow_demo auto_follower
    $ ros2 run control_demo robot_control (串口名需与虚拟串口对应，如/dev/pts/0)
    ```

    `$ xxd /dev/pts/2 (串口名需与另一个虚拟串口对应)`

## 12.6 改进与思考

当前系统为基础版自动跟随算法，存在以下可优化方向：

+ 利用ROS2参数机制，对其中的阈值、系数等参数化
+ 速度控制震荡，可引入PID控制代替比例控制
+ 光照影响颜色检测精度，可采用更稳定的感知算法
