# 8. Launch文件

在复杂的机器人系统中，一个完整的功能往往需要多个节点协同工作。例如，实现自主导航可能需要同时启动激光雷达驱动节点、定位节点、路径规划节点和控制节点。如果通过手动逐个输入ros2 run命令启动这些节点，不仅操作繁琐，还容易因参数遗漏或启动顺序错误导致系统异常。

ROS2 提供的**Launch文件**（启动文件）正是为解决这一问题而设计的工具。它允许开发者通过一个文件定义多个节点的启动参数、运行顺序、依赖关系及配置信息，实现多节点的自动化启动与管理。

本章将介绍Launch文件的作用、语法规范、编写及使用方法。

## 8.1 Launch文件介绍

### 8.1.1 什么是Launch文件？

Launch 文件是一个描述性配置文件，用于定义节点的启动参数、环境变量、命名空间及节点间的依赖关系。其核心作用是自动化多节点启动流程，替代手动执行多个ros2 run命令的重复操作。

例如，启动turtlesim仿真节点以及速度发布节点，手动操作需要两个终端分别执行：

```
# 终端1
ros2 run turtlesim turtlesim_node
# 终端2
ros2 run turtle_demo turtle_vel_pub
```

而通过 Launch 文件，只需一个命令即可启动所有节点。

### 8.1.2 Launch 文件的优势

相比手动启动节点，Launch 文件具有以下核心优势：

+ 简化操作：单次命令启动多个节点，无需手动管理多个终端；
+ 参数集中配置：在一个文件中统一设置所有节点的参数，避免重复输入；
+ 控制启动顺序：定义节点间的依赖关系（如 "定位节点必须在雷达节点启动后启动"）；
+ 命名空间隔离：为节点或参数添加命名空间，避免多实例运行时的命名冲突；
+ 动态逻辑支持：通过 Python 编写的 Launch 文件可实现条件判断、循环等逻辑（如根据参数决定是否启动调试节点）；
+ 可重用性：通过 "包含" 机制复用其他 Launch 文件，实现模块化配置（如导航功能复用传感器启动配置）。

### 8.1.3 Launch 文件格式
ROS2 支持三种 Launch 文件格式，各有适用场景：

+ **Python 格式**（推荐）：以.py为后缀，通过 ROS2 提供的 Python API 定义启动逻辑，支持条件判断、变量运算等复杂逻辑，是 ROS2 官方推荐的格式；

+ **XML 格式**：以.launch.xml为后缀，采用标签式语法，结构清晰但灵活性较低，适合简单的静态配置；

+ **YAML 格式**：以.launch.yaml为后缀，采用键值对语法，简洁易读但功能有限，仅支持基础配置。

下面，我们将以启动以下两个节点为例，演示**Python 格式**的Launch文件基本用法。

+ 节点1：`turtlesim`功能包下的`turtlesim`仿真节点。
+ 节点2：`turtle_demo`功能包下的`turtle_vel_pub`速度发布节点。（`turtle_demo`功能包已在第5章创建）

## 8.2 编写Launch文件

Launch文件通常放在某个功能包的名为`launch`的目录下，可以启动本功能包和其他功能包中的节点。因此，我们在前面的`turtle_demo`功能包下创建名为`launch`的文件夹。

```
cd ~/ros2_ws/src/turtle_demo
mkdir launch
```

在`launch`目录下创建`turtle_launch.py`文件：

```
cd launch
gedit turtle_launch.py
```

内容如下：

```
# 导入Launch文件必备模块
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    # 启动海龟仿真节点
    turtle_node = Node(
        package='turtlesim',  # 节点所属功能包
        executable='turtlesim_node',  # 节点可执行文件名
        name='my_turtle',  # 节点运行时的名称（覆盖默认名称）
        output='screen',  # 节点日志输出到终端（默认仅输出到日志文件）
    )

    # 启动速度发布节点
    vel_pub_node = Node(
        package='turtle_demo', # 节点所属功能包
        executable='turtle_vel_pub', # 节点可执行文件名
        name='vel_pub', # 节点运行时的名称（覆盖默认名称）
        output='screen', # 节点日志输出到终端（默认仅输出到日志文件）
    )

    # 返回Launch描述对象，包含所有要启动的动作
    return LaunchDescription([
        turtle_node,  # 先启动仿真节点
        vel_pub_node,   # 再启动速度发布节点
    ])
```

**代码解析：**

+ Node：表示启动的节点，关键参数包括：
    + package：节点所在的功能包名称；
    + executable：节点的可执行文件名；
    + name：可选，指定节点运行时的名称，若不设置则使用可执行文件默认名称；
    + output：可选，设置日志输出方式，screen表示终端输出，log表示仅日志文件。
+ LaunchDescription：Launch 文件的核心对象，用于存储所有需要执行的启动动作（如节点启动、环境变量设置等）；

## 8.3 启动Launch文件

### 8.3.1 直接启动

可以使用`ros2 launch <文件名>`的方式，直接启动`turtle_launch.py`文件（需要确保当前目录为此Launch文件所在目录）：

```
ros2 launch turtle_launch.py
```

此时，海龟仿真节点和速度发布节点都启动了，海龟会按照发布的速度运动。

### 8.3.2 通过功能包启动

直接启动时，需要切换到Launch文件的具体路径，比较麻烦。大多数情况是通过`ros2 launch <功能包名> <文件名>`命令启动。因此，为了让ROS2系统知道`turtle_demo`功能包下有这个Launch文件，需要打开功能包根目录的`setup.py`文件（`~/ros2_ws/src/turtle_demo/setup.py`），在最开始导入`os`和`glob`库，并找到`data_files`部分，添加Launch文件的路径。

```
...
import os  # 新增导入os库
from glob import glob # 新增导入glob库
...
    data_files=[
        ('share/ament_index/resource_index/packages',
            ['resource/' + package_name]),
        ('share/' + package_name, ['package.xml']),
        # 新增这一行：告诉ROS2 launch目录下的.py文件是Launch文件
        (os.path.join('share', package_name, 'launch'), glob('launch/*.py')),
    ],
...
```

配置完`setup.py`文件后，先构建：

```
cd ~/ros2_ws
colcon build
```

然后可以在任意位置，通过`ros2 launch <功能包名> <文件名>`命令启动Launch文件：

```
ros2 launch turtle_demo turtle_launch.py
```

## 8.4 Launch节点参数

除了启动节点，Launch 文件还能直接给节点设置参数（比如海龟窗口的背景色），不用手动用`ros2 param set`修改了。

打开`turtle_launch.py`文件，为海龟仿真节点设置背景色参数：

```
# 导入Launch文件必备模块
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    # 启动海龟仿真节点
    turtle_node = Node(
        package='turtlesim',  # 节点所属功能包
        executable='turtlesim_node',  # 节点可执行文件名
        name='my_turtle',  # 节点运行时的名称（覆盖默认名称）
        output='screen',  # 节点日志输出到终端（默认仅输出到日志文件）
        # 新增：设置参数（背景色为红色）
        parameters=[
            {'background_r': 255},  # 红色分量（0-255）
            {'background_g': 0},    # 绿色分量
            {'background_b': 0}     # 蓝色分量
        ],
    )

    # 启动速度发布节点
    vel_pub_node = Node(
        package='turtle_demo', # 节点所属功能包
        executable='turtle_vel_pub', # 节点可执行文件名
        name='vel_pub', # 节点运行时的名称（覆盖默认名称）
        output='screen', # 节点日志输出到终端（默认仅输出到日志文件）
    )

    # 返回Launch描述对象，包含所有要启动的动作
    return LaunchDescription([
        turtle_node,  # 先启动仿真节点
        vel_pub_node,   # 再启动速度发布节点
    ])
```

修改完`turtle_launch.py`文件后，需要先构建，然后再运行：

```
cd ~/ros2_ws
colcon build
ros2 launch turtle_demo turtle_launch.py
```

此时海龟窗口的背景为红色，参数在启动时就生效了。
